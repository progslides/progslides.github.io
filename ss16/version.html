Vorteile Versionskontrolle

  • Speichert sich alle Versionen von euren Dateien
  • Dateien liegen auf Server - kein Lästiges hin und herschicken von Code per Email
  • Wer hat denn das geschrieben? - GIT/SVN weiß es!
  • Was wurde schon implementiert/in welcher Version funktionieren welche Features? - GIT/SVN Log!

=> GIT/SVN sind Super Hilfreiche Tools, die jeder Programmierer verwenden sollte!

Verwendung wird in späteren LVs Pflicht sein!

Vorteile GIT vs SVN

GIT

  • Super Branch/Tag Support
    • Jeder Nutzer kann auf einem eigenen Branch arbeiten, und "merged" erst wenn sein Code funktioniert diesen in den Master
    • Besondere Versionen (z.B. "Commandline Funktioniert" oder "1. Submission") können mit Tags versehen werden.
  • Lokales arbeiten und commiten (Versionen abspeichern) ist auch ohne Internetverbindung möglich

SVN

  • Mit Tortoise SVN integration in den Windows Explorer
  • Weniger Schritte notwendig, um einen Stand auf den Server zu bringen

=> Wir empfehlen die Verwendung von GIT.

Oh Gott, was ist denn ein SSH-Key? :/

Authetifizierungsmöglichkeit für SSH (Secure Shell)

  • Kann mit PuTTYgen oder ssh-keygen in der Shell erstellt werden
  • Besteht aus einem Private-Key und einem Public-Key
  • Der PUBLIC Key wird auf git.tugraz.at hochgeladen
  • Der PRIVATE Key liegt bei euch und wird verwendet, um sich beim Server zu authentifizieren.

Ich hab ein Repository und einen SSH-Key hochgeladen, was mache ich jetzt?

  • git clone >repository-url< in der Shell
  • ODER
  • Nutzen einer der vielen GIT-GUIs, und "Clone" UI nutzen:
    • SmartGit
    • SourceTree
    • gitg

Nächste Schritte

  • Erstelle deine Dateien
  • git add >filename(s)< (Diese Files gehören zu meiner neuen Version)
  • Oder in GUI auswahl der Files und oder bestimmter Zeilen und "Stage"
  • git commit -m ">message<" Um eine neue Version zu erstellen. Hilfreiche Messages enthalten:
    • Welche Features sind in der Neuen Version dazugekommen?
    • Welche Funktionen wurden geändert?
    • => Optimaler Weise ein Commit pro Änderung
  • git pushSobald die Versionen auf den Server hochgeladen werden sollen
